Preprocessing

Ways to implement preprocessing.

LESS, for instance, could be implemented through an extension in your code editor. It could also be implemented by downloading a .js file and including it in the header after including the .less file. To run LESS on the server side, you would need to install the necessary components on your server.

Mechanisms used in LESS.

  • Variables are used to specify multiple values for a CSS element in one location. For example, if you find that H3, UL and OL all share the exact same properties, this allows you to list those properties in one location then simply reference that location.
  • Mixins allow you to setup a class with multiple properties and then reference that class when setting up other classes or rulesets.
  • Nesting helps you organize multiple classes or IDs in a concise structure. This could be used when creating a navigation bar. Instead of having multiple rules that cover each scenario in a navbar (active, hover...), you can create one master class then cascade the rules beneath that. Nested At-Rules and Bubbling provides a way to organize rules like media queries.
  • Operations allows for arithmetical operations such as plus, minus, multiplication and division. They can operate on any number, color or variable.
  • Functions can manipulate things like colors and HTML elements aspects in style sheets. They can perform math to enhance the properties in the style sheet.
